# 🗳️ 투표 모듈
{: .no_toc }

시민이 함께 결정하기 위한 다양한 투표 방식. 단순히 한 번 결정하고 끝나는 의사결정이 아니라, 대화와 정보 속에서 의견이 변화하는 과정을 시각화하고, 시민이 언제든 마음을 바꿀 수 있는 살아있는 투표를 추구합니다.

{: .no_toc .text-delta }

1. TOC

---

## 1. 찬반 투표

✅ 구현 완료
{: .label .label-green }

찬성과 반대 중 하나를 선택하는 가장 기본적인 투표 방식.

## 2. 객관식 투표 (설문형)

✅ 구현 완료
{: .label .label-green }

단일 또는 복수 선택으로 의견을 수렴하는 설문형 투표.

## 3. 점수 투표

✅ 구현 완료
{: .label .label-green }

여러 선택지에 점수를 매겨 선호도를 표현하고 우선순위를 도출하는 투표.

## 4. 은하투표

✅ 구현 완료
{: .label .label-green }
⭐ 빠띠 오리지널
{: .label .label-purple }

시민 의견 분포를 별무리 형태로 시각화하는 빠띠 자체 개발 투표 방식. 단순한 찬반을 넘어 의견 지형을 드러내고, 별별대화·한국의 대화 등 숙의 매칭의 출발점이 됩니다.

## 5. 배틀 투표

✅ 구현 완료
{: .label .label-green }
⭐ 빠띠 오리지널
{: .label .label-purple }

대립하는 두 주장을 두고 참가자가 실시간으로 선호를 선택하는 토론형 투표. 의견의 쏠림과 변화가 시각적으로 드러납니다.

## 6. 실시간 투표 변경

✅ 구현 완료
{: .label .label-green }
⭐ 빠띠 오리지널
{: .label .label-purple }

대부분의 투표는 한 번 결정하면 끝납니다. 빠띠의 토론형 투표는 다릅니다. 시민이 토론과 대화 속에서 언제든 자신의 선택을 바꿀 수 있고, 전체의 의견 분포가 어떻게 움직이는지를 함께 봅니다. 빠띠가 추구하는 숙의 민주주의의 핵심 기능.

## 7. 확정형 투표

✅ 구현 완료
{: .label .label-green }

최종 의사결정용으로, 투표 버튼을 눌러야 표가 확정되는 방식. 회의·총회·임원 선출 등 결과를 봉인해야 할 때 사용합니다.

## 8. 비밀(블라인드) 투표

✅ 구현 완료
{: .label .label-green }

스태프와 개설자조차 개별 결과를 알 수 없는 완전 익명 투표.

## 9. 전자서명

✅ 구현 완료
{: .label .label-green }

의사결정과 청원에 법적 효력의 서명을 받는 기능. 안내문 작성과 서명 파일 다운로드를 지원합니다.

## 10. 대규모 전자투표

✅ 구현 완료
{: .label .label-green }

1만 명 이상이 참여하는 대규모 온라인 투표를 안정적으로 지원. 주민참여예산, 시민 총회, 대규모 시민의회에 사용됩니다.

## 11. 쿼드라틱 투표

⬜ 향후 과제
{: .label }

의견 강도에 비례해 투표 비용이 제곱으로 늘어나는 가중 투표 방식. 다수의 약한 선호와 소수의 강한 선호 사이의 균형을 만듭니다.
